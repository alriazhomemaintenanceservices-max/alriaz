1. Critical SEO Issues

Critical Homepage & key pages have no title tag or meta description

7 pages are 'use client' components with no server-side metadata: the homepage, /services/, /services/electrician/, /services/plumber/, /services/intercom/, /about-us/, and /contact/. They ship with no <title>, no meta description, no canonical, and no Open Graph.

A prewritten src/app/metadata.ts exists but is never imported — it's orphaned. Your single most important page (the homepage) currently has no title in Google's eyes.

Fix: Add a server layout.tsx next to each client page that exports metadata (or convert these pages to a server wrapper + client child, exactly like the area pages already do). Wire up or delete the orphaned metadata.ts.

Critical English version is completely invisible to search engines

Language is a client-side Zustand toggle; the URL never changes (no /en/ path) and <html lang="ar"> is hard-coded. Google only ever sees the Arabic content. There are no hreflang tags, so the entire English audience is unreachable via organic search.

Fix: Either accept Arabic-only for SEO (and drop the EN claim), or build real /en/ routes with hreflang alternates linking the AR ↔ EN versions.

Critical No Open Graph image — broken WhatsApp / social previews

No og:image is set anywhere. Since your entire funnel pushes people to share and message on WhatsApp, every shared link shows a blank, unappealing preview — directly hurting the channel you rely on most.

Fix: Add a branded 1200×630 og:image (logo + "كهربائي وسباك · خلال ساعة · 24/7" + phone) in the root metadata, plus twitter:card.

Critical Conflicting robots files

Both public/robots.txt (blocks /api/ /_next/ /studio/) and src/app/robots.ts (blocks /private/) exist. The static file in public/ wins, so the generated robots.ts is silently ignored — and the two disagree, which is a maintenance trap.

Fix: Keep one. Delete public/robots.txt and let robots.ts be the single source (it already points to the sitemap).

2. High-Priority Issues

High Canonical vs. trailing-slash mismatch

next.config.ts sets trailingSlash: true and the sitemap emits URLs with a slash (/كهربائي-yasmin/), but the area-page canonicals are written without it (/كهربائي-yasmin). Google receives contradictory signals about the canonical URL.

Fix: Add the trailing slash to every canonical (and legal-page canonicals) so it matches the sitemap and the live 308 redirect.

High Fabricated review rating in structured data

layout.tsx injects aggregateRating 4.9 / 127 reviews globally, but there are no corresponding on-page reviews with Review schema to back it. Unsubstantiated aggregateRating is a known trigger for Google manual actions / structured-data penalties.

Fix: Only mark up ratings that are real and visible on the page, with individual Review items. Otherwise remove aggregateRating.

High No custom 404 / error / loading pages

There is no not-found.tsx, error.tsx, or loading.tsx anywhere in src/app. Mistyped Arabic URLs and broken backlinks hit the bare default 404 with no navigation back into the funnel.

Fix: Add a branded not-found.tsx (with call/WhatsApp CTAs + area links) and a root error.tsx.

High Thin content & no blog — weak topical authority

Zero article/blog pages. For a local lead-gen site, informational content ("كم تكلفة تصليح المكيف في الرياض؟", "علامات تسريب المياه") is how you capture top-of-funnel search and build authority that lifts the money pages. Competitors that rank do exactly this.

Fix: Add a /blog/ (or /مقالات/) section with 8–12 problem/cost/how-to articles, internally linking to the relevant service + area pages.

4. What's Already Done Well

Good Area pages are properly optimized

All 54 area pages use a server wrapper that exports unique title, description, canonical, and Open Graph, and the shared component injects Service, FAQPage, and BreadcrumbList JSON-LD. This is solid programmatic local SEO.

Good Global structured data & dynamic sitemap

layout.tsx ships Organization + LocalBusiness + WebSite JSON-LD on every page. sitemap.ts is a clean single source of truth covering all 66 URLs. Security headers (CSP, X-Frame-Options, nosniff, Referrer-Policy) are configured.

Good Arabic-URL proxy handled correctly

proxy.ts cleanly rewrites encoded Arabic URLs to ASCII route folders while preserving the Arabic address bar, and enforces the trailing slash with a 308. This is the tricky part of the setup and it's done right.
